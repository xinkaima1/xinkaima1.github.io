## 1. 仓库与部署基础

### 仓库结构说明

```
xinkaima1.github.io/
├── _config.yml              # 全站配置文件（站点名称、头像、社交链接等）
├── _pages/                 # 页面文件目录
│   ├── about.src.md        # 首页源文件（中英文合一，只需维护此文件）
│   ├── projects.md         # 项目列表页
│   ├── cv.md              # 在线简历页
│   └── news.md            # 动态/新闻页
├── _projects/              # 项目经历文件目录
│   ├── 1_craic_lunar.md  # 项目1：月球探测
│   └── 2_craic_simuro.md # 项目2：Simuro足球
├── _data/
│   └── socials.yml        # 社交链接配置（GitHub/邮箱等）
├── _bibliography/
│   └── papers.bib         # 论文 BibTeX 数据
├── assets/
│   ├── img/               # 图片存放目录（头像、项目截图等）
│   └── pdf/               # PDF 文件目录（CV 下载等）
├── split_lang.py           # 中英文页面自动生成脚本
└── .github/workflows/
    └── deploy.yml          # 自动部署工作流
```

### 自动部署原理

本仓库启用了 **GitHub Pages** 自动部署：

1. 你 `push` 代码到 `main` 分支
2. GitHub Actions 自动运行 `deploy.yml` 工作流
3. 工作流先执行 `split_lang.py` 生成中英文页面
4. 然后 Jekyll 构建静态站点
5. 最后部署到 `gh-pages` 分支，约 **3~5 分钟**后生效

> ⚠️ **无需本地安装 Jekyll**，所有构建都在 GitHub 云端完成。

## 2. 修改首页 About 内容

### 核心文件

| 文件 | 说明 |
|------|------|
| `_pages/about.src.md` | **唯一需要编辑的文件**（中英文合一源文件） |

> ⚠️ **不要直接编辑** `_pages/about.md` 和 `_pages/about-zh.md`，这两个文件是由 `split_lang.py` 脚本自动生成的。

### 编辑方式

用任意文本编辑器（推荐 VS Code / Notepad++）打开 `_pages/about.src.md`，按以下规则修改：

```markdown
## About Me
[EN] Hello! I'm **Xinkai Ma**...      ← 英文版显示此行
[ZH] 你好！我是**马鑫凯**...            ← 中文版显示此行

[EN] I'm currently studying...          ← 英文版
[ZH] 我就读于...                       ← 中文版

（无 [EN]/[ZH] 标记的行在两个版本都显示）
```

**标记规则：**

| 写法 | 效果 |
|------|------|
| `[EN] 内容` | 仅出现在英文版 `about.md` |
| `[ZH] 内容` | 仅出现在中文版 `about-zh.md` |
| `内容`（无标记） | 两个版本都保留 |

### 修改步骤

1. 打开 `_pages/about.src.md`
2. 找到要修改的段落，按需修改 `[EN]` 或 `[ZH]` 后的内容
3. 保存文件
4. `git add .` → `git commit -m "更新自我介绍"` → `git push`
5. 等待 3~5 分钟，刷新首页查看效果

### 预览切换效果

- 英文首页：`https://xinkaima1.github.io/`
- 中文首页：`https://xinkaima1.github.io/about-zh/`
- 页面右上角有浮动按钮 `🇨🇳 中文` / `🇺🇸 EN` 可一键切换

## 3. 修改个人基础信息

### 修改站点标题和姓名

编辑 `_config.yml`：

```yaml
# 第 5~8 行左右
title: blank          # 留空则显示全名；填写则显示自定义标题
first_name: Xinkai
middle_name:
last_name: Ma
```

### 修改个人头像

1. 将新头像图片命名为 `prof_pic.jpg`（建议正方形，≥400×400px）
2. 放入 `assets/img/` 目录，覆盖原文件
3. 如果文件名不同，修改 `_pages/about.src.md` 的 front matter：

```yaml
profile:
  image: prof_pic.jpg   # ← 改成你的文件名
  image_circular: false  # true = 圆形头像，false = 方形头像
```

### 修改教育背景（profile 信息栏）

编辑 `_pages/about.src.md` 的 front matter（`---` 之间的内容）：

```yaml
profile:
  more_info: >
    [EN] <p>College of Biological and Agricultural Engineering</p>
    [EN] <p>Jilin University</p>
    [EN] <p>Changchun, Jilin, China</p>

    [ZH] <p>吉林大学</p>
    [ZH] <p>中国, 吉林, 长春</p>
```

### 修改社交链接

编辑 `_data/socials.yml`，按需增删：

```yaml
- name: GitHub
```

## 4. 导航栏菜单配置

### 导航栏工作原理

al-folio 的导航栏菜单由 **每个页面的 front matter 中的 `nav` 字段**控制。

在任意 `_pages/*.md` 文件中，设置：

```yaml
---
layout: page
title: projects       # 菜单显示的文字
permalink: /projects/ # 页面路径
nav: true            # ← true = 显示在导航栏；false = 不显示
nav_order: 3          # ← 菜单排列顺序（数字越小越靠左）
---
```

### 修改菜单名称

直接修改对应页面文件的 `title` 字段，例如把 "projects" 改成 "项目"：

```yaml
# _pages/projects.md
title: 项目            # 导航栏显示"项目"
```

### 调整菜单顺序

修改 `nav_order` 数值，从小到大排列：

| nav_order | 菜单项 |
|-----------|--------|
| 1 | About |
| 2 | Projects |
| 3 | Blog（如有）|
| 4 | CV |

### 新增导航菜单项

1. 在 `_pages/` 下新建 `.md` 文件（如 `teaching.md`）
2. 在 front matter 中设置 `nav: true` 和 `nav_order`
3. 保存并 push，自动部署后生效

### 隐藏某个菜单项

将对应页面文件的 `nav` 改为 `false`，或删掉 `nav` 字段。

## 5. 添加项目经历

### `_projects/` 目录作用

所有项目经历以独立 `.md` 文件存放在 `_projects/` 目录下。
每个文件对应 Projects 页面的一张卡片。

### 新建项目文件

1. 在 `_projects/` 下新建文件，命名格式：`3_项目名.md`（数字越大排序越靠后）

2. 文件头部（front matter）**必填参数**：

```yaml
---
layout: page           # 固定值，不要改
title: 项目中文标题     # 项目卡片标题
description: 简短描述   # 项目卡片副标题
img:                    # 项目封面图（放在 assets/img/ 下）
importance: 1           # 排序权重（数字越小越靠前）
category: work          # 项目分类：work=科研/竞赛，fun=兴趣项目
related_publications: false  # 是否关联论文：false 或论文 key
---
```

3. 文件正文（front matter 下方）写项目详细介绍，支持 Markdown 格式。

### 完整示例

```markdown
---
layout: page
title: CRAIC Lunar Exploration
description: Autonomous lunar rover navigation and exploration system
img:
importance: 1
category: work
related_publications: false
---

## CRAIC Lunar Exploration Project

**中国机器人人工智能大赛**月球探测项目...

### Project Overview

- 🗺️ **Autonomous Navigation** - 路径规划与 obstacle avoidance
- 🔍 **Environment Perception** - 传感器融合

### Technologies

- **ROS** - Robot Operating System
- **SLAM Algorithms** - 同步定位与建图
```

### 添加项目封面图

1. 将图片放入 `assets/img/` 目录
2. 在 front matter 中填写图片文件名：

```yaml
img: project_cover.jpg   # 对应 assets/img/project_cover.jpg
```

> 如果 `img` 留空，项目卡片将不显示封面图。

### 设置项目分类

编辑 `_pages/projects.md` 中的 `display_categories`：

```yaml
display_categories: [work]   # 只显示 category: work 的项目
```

可选值：
- `work` — 科研/竞赛项目（当前已启用）
- `fun` — 兴趣/个人项目

如果想同时显示两个分类，改为：

```yaml
display_categories: [work, fun]
```

### 让新项目在 Projects 页面正常展示

**检查清单：**

- [ ] 文件放在 `_projects/` 目录下
- [ ] 文件扩展名是 `.md`
- [ ] front matter 中有 `category: work`（或 `fun`）
- [ ] `_pages/projects.md` 的 `display_categories` 包含该分类
- [ ] 已 push 到 GitHub 并等待部署完成

## 6. CV 在线简历维护

### CV 页面文件

| 文件 | 说明 |
|------|------|
| `_pages/cv.md` | CV 页面配置（布局、PDF 路径等）|
| `_data/cv.yml` | CV 详细内容（教育经历、获奖、技能等）|

### 编辑 CV 内容

打开 `_data/cv.yml`，按已有格式修改对应栏目：

```yaml
name: Xinkai Ma
email: your-email@xx.com
phone: +86 1xx-xxxx-xxxx
website: https://xinkaima1.github.io

education:
  - degree: B.Eng. (expected)
    institution: Jilin University
    year: 2023 - 2026
    description: ...

awards:
  - title: 奖项名称
    date: 2025
    description: ...

skills:
  - category: Programming
    items: Python, C++, MATLAB
```

### 配置 PDF 下载

1. 将 CV 的 PDF 文件放入 `assets/pdf/` 目录
2. 编辑 `_pages/cv.md` 修改路径：

```yaml
cv_pdf: /assets/pdf/your_cv.pdf   # ← 改成你的 PDF 文件名
```

### CV 格式选择

`_pages/cv.md` 支持两种渲染格式：

```yaml
cv_format: rendercv   # 推荐：RenderCV 风格，美观专业
# 或
cv_format: jsonresume  # JSON Resume 风格
```

## 7. 本地修改与部署流程

### 完整操作流程

```bash
# 1. 进入仓库目录
cd D:/桌面/github/xinkaima1.github.io

# 2. 拉取最新代码（首次或多人协作时必做）
git pull

# 3. 用编辑器修改文件（见前面各章节）

# 4. 查看修改内容
git status
git diff

# 5. 暂存并提交
git add .
git commit -m "描述本次修改内容"

# 6. 推送到 GitHub（触发自动部署）
git push

# 7. 等待 3~5 分钟，访问网站查看效果
```

### commit 信息规范

建议使用中文简短描述，例如：

```
git commit -m "新增机器人竞赛项目"
git commit -m "更新个人简介中英文版本"
git commit -m "修改导航栏顺序"
git commit -m "替换个人头像为高清版"
```

### 部署状态查看

1. 打开 [https://github.com/xinkaima1/xinkaima1.github.io/actions](https://github.com/xinkaima1/xinkaima1.github.io/actions)
2. 查看最新一条 workflow 的运行状态
3. 绿色 ✅ = 部署成功；红色 ❌ = 失败（点击查看错误日志）

## 8. 常见问题排查

### 项目页面空白 / 项目不显示

**原因 1：** `category` 字段与 `display_categories` 不匹配

→ 检查项目文件的 `category` 是否在 `_pages/projects.md` 的 `display_categories` 列表中

**原因 2：** 文件名或路径错误

→ 确认文件在 `_projects/` 目录下，且扩展名为 `.md`

**原因 3：** front matter 格式错误（缺少冒号、缩进不对）

→ 检查 YAML 格式，确保冒号后有空格，缩进使用空格（不用 Tab）

### 首页 404 / 页面打不开

**原因：** `permalink` 路径配置错误

→ 检查对应页面文件的 `permalink` 字段，确保以 `/` 开头和结尾，例如：

```yaml
permalink: /about-zh/   # ✅ 正确
permalink: about-zh       # ❌ 错误（缺少前后斜杠）
```

### 图片加载失败 / 图片不显示

**原因 1：** 图片路径错误

→ 确认图片已放入 `assets/img/` 目录，且文件名完全匹配（区分大小写）

**原因 2：** 文件名含中文或空格

→ 图片文件名使用英文、数字、下划线，不含空格和中文

### 修改后网站没变化

**原因 1：** 部署还未完成

→ 等待 3~5 分钟，或查看 [Actions 页面](https://github.com/xinkaima1/xinkaima1.github.io/actions) 确认部署状态

**原因 2：** 浏览器缓存

→ 按 `Ctrl + F5`（Windows）或 `Cmd + Shift + R`（Mac）强制刷新

**原因 3：** 修改了错误文件

→ 记住：`about.md` 和 `about-zh.md` 是自动生成的，**不要直接编辑**，请改 `about.src.md`

### 中英文切换按钮不显示

**原因：** `language-switcher.js` 未正确注入

→ 检查 `.github/workflows/deploy.yml` 中是否有 `Inject language switcher script` 步骤，
   或手动验证网站上是否有 `<script src="/assets/js/language-switcher.js"></script>` 标签

### YAML 格式报错（部署失败）

GitHub Actions 失败时，常见 YAML 错误：

| 错误 | 正确写法 |
|------|----------|
| `title:关于我` | `title: 关于我`（冒号后要有空格）|
| 用 Tab 缩进 | 用空格缩进（建议 2 个空格）|
| 描述文字含冒号 `:` | 用引号包裹：`description: "包含: 冒号的文字"` |
