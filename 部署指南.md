## 🚀 部署步骤

### 第一步：创建 GitHub 仓库

1. 登录你的 GitHub 账号（xinkaima1）
2. 创建一个新仓库，仓库名必须是：`xinkaima1.github.io`
   - 注意：仓库名必须和你的用户名完全一致，后缀是 `.github.io`
   - 选择 Public（公开）
   - 不需要初始化 README

### 第二步：上传文件

**方法一：直接上传（推荐新手）**

1. 打开你刚创建的 `xinkaima1.github.io` 仓库
2. 点击 "Add file" → "Upload files"
3. 把这个文件夹里的**所有文件和文件夹**都拖进去
4. 点击 "Commit changes"

**方法二：Git 命令行（推荐有经验的）**

```bash
# 进入项目目录
cd xinkaima1.github.io

# 初始化 git
git init
git add .
git commit -m "Initial commit"

# 添加远程仓库（替换成你的仓库地址）
git remote add origin https://github.com/xinkaima1/xinkaima1.github.io.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

### 第三步：启用 GitHub Pages

1. 进入仓库的 **Settings**（设置）
2. 左侧菜单找到 **Pages**
3. 在 **Build and deployment** 部分：
   - Source 选择 `Deploy from a branch`
   - Branch 选择 `main` 分支，文件夹选择 `/ (root)`
4. 点击 **Save**

### 第四步：等待部署

- GitHub Pages 会自动构建你的网站（通常需要 1-3 分钟）
- 部署完成后，你就可以通过 `https://xinkaima1.github.io` 访问了
- 可以在仓库的 **Actions** 页面查看部署进度

## 🎨 后续自定义

### 1. 替换头像照片

把你的照片命名为 `prof_pic.jpg`，放到 `assets/img/` 目录下，替换原来的文件。

**建议：**
- 尺寸：正方形（比如 400x400 像素）
- 格式：JPG 或 PNG
- 文件大小不要太大（建议小于 500KB）

### 2. 修改个人简介

编辑 `_pages/about.md` 文件，可以用 Markdown 格式写你的介绍。

### 3. 添加更多项目

在 `_projects/` 目录下新建 `.md` 文件，参考已有的项目格式。

### 4. 添加 CV / 简历

把你的 PDF 简历放到 `assets/pdf/` 目录，然后在 `_data/socials.yml` 里取消 `cv_pdf` 的注释。

### 5. 添加新闻/动态

在 `_news/` 目录下新建文件，可以展示你的最新动态。

## ❓ 常见问题

**Q: 网站部署后打不开？**
- 检查仓库名是否正确（必须是 `用户名.github.io`）
- 检查 GitHub Pages 设置是否正确
- 等待几分钟，部署需要时间

**Q: 样式显示不对？**
- 检查 `_config.yml` 里的 `url` 和 `baseurl` 是否正确
- 你的情况：`url: https://xinkaima1.github.io`，`baseurl` 留空

**Q: 怎么添加更多页面？**
- 在 `_pages/` 目录下新建 `.md` 文件即可
- 参考模板里的其他页面格式
